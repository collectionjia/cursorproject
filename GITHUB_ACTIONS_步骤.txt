GitHub Actions 云端打包 macOS 版 — 操作步骤
==========================================

前提
----
- 没有 Mac 也可以，用 GitHub 提供的免费 macOS 虚拟机
- 需要 GitHub 账号（免费注册：https://github.com/signup）
- 需要官方原版 macOS 安装包（.app），并打成 zip

────────────────────────────────────────
第一步：准备原版 macOS 安装包（支持 .dmg 或 .zip）
────────────────────────────────────────

1. 获取官方 AIAssistant-360 macOS 版（两个架构各一份）：
   - Apple 芯片 (M1/M2/M3…)：arm64
   - Intel Mac：x86_64

2. 官方若是 .dmg，可直接使用，无需手动转 zip。
   dmg 里通常结构为：
   AIAssistant-360-arm64.dmg
   └── AIAssistant-360.app

   若是 .zip 也可以：
   AIAssistant-360-arm64.zip
   └── AIAssistant-360.app

3. 把 .dmg 或 .zip 放到能直接下载的地址：
   A) 网盘「分享直链」（百度/阿里云/123 等，需 curl 能直接下到 zip）
   B) GitHub Release（见下方「附：用 GitHub Release 托管 zip」）
   C) 自己的服务器 HTTPS 链接

测试直链是否可用（在 Windows PowerShell）：
  curl -I "你的zip地址"
  应返回 HTTP 200，且 Content-Type 含 zip 或 octet-stream

────────────────────────────────────────
第二步：创建 GitHub 仓库并上传代码
────────────────────────────────────────

1. 打开 https://github.com/new
2. 仓库名例如：aiassistant-360-patch
3. 选 Private（私有，推荐）或 Public
4. 不要勾选「Add README」（保持空仓库）
5. 点 Create repository

6. 在本机打开 PowerShell，执行：

   cd d:\project\cursor_project
   .\init_github_repo.ps1 -RepoUrl "https://github.com/你的用户名/aiassistant-360-patch.git"

   （把 RepoUrl 换成你刚创建的仓库地址）

7. 浏览器打开仓库 → 应能看到：
   - .github/workflows/build-macos.yml
   - build_macos.py
   - patch_no_renew_menu.py
   - pyinstxtractor.py

────────────────────────────────────────
第三步：运行 GitHub Actions 打包
────────────────────────────────────────

1. 打开仓库页面 → 顶部「Actions」
2. 左侧选「Build macOS Patched App」
3. 右侧「Run workflow」
4. 填写（支持 .dmg 或 .zip 直链）：
   - arm64_zip_url：Apple 芯片原版 dmg/zip 直链（必填）
   - x86_zip_url：Intel 原版 dmg/zip 直链（两个都要就打填上）
5. 点绿色「Run workflow」
6. 等待约 5～15 分钟（两个 job 可并行）

────────────────────────────────────────
第四步：下载打包结果
────────────────────────────────────────

1. 进入刚跑完的那次 workflow 运行记录
2. 页面底部 Artifacts：
   - AIAssistant-360-no-renew-arm64.zip
   - AIAssistant-360-no-renew-x86_64.zip（若填了 Intel 地址）
3. 下载 zip，解压得到 .app，拷到 Mac 使用

Mac 上若提示无法打开：
  xattr -dr com.apple.quarantine AIAssistant-360-no-renew-arm64.app

────────────────────────────────────────
附：用 GitHub Release 托管原版 zip（推荐）
────────────────────────────────────────

1. 仓库 → Releases → Create a new release
2. Tag：v1.0-original-arm64
3. 上传 AIAssistant-360-arm64.zip
4. Publish release
5. 在 release 页面，右键 zip → 复制链接地址
   格式类似：
   https://github.com/用户/仓库/releases/download/v1.0-original-arm64/AIAssistant-360-arm64.zip
6. 把这个链接填到 workflow 的 arm64_zip_url

Intel 版同理，tag 如 v1.0-original-x86_64

────────────────────────────────────────
常见问题
────────────────────────────────────────

Q: Actions 里找不到 workflow？
A: 确认已 push .github/workflows/build-macos.yml，且默认分支是 main 或 master。

Q: 下载 zip 失败 No .app found？
A: dmg/zip 里必须包含 .app；dmg 会自动挂载提取，无需你手动转换。

Q: 只有 dmg 没有 zip 可以吗？
A: 可以，直接把 dmg 下载直链填到 workflow 输入框即可。

Q: 私有仓库 Release 的 zip，Actions 能下载吗？
A: 同一仓库内 Release 资源是公开的下载链接，一般可直接用。

Q: 免费额度够吗？
A: 私有仓库每月约 2000 分钟 macOS 额度，打一两次足够。
